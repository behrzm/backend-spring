Привет! Я интегрировал Android клиент с backend сервером. Вот ЧТО НУЖНО СДЕЛАТЬ:

═══════════════════════════════════════════════════════════════════════════════
✅ ШАГ 1: ЗАПУСТИТЕ BACKEND
═══════════════════════════════════════════════════════════════════════════════

1.1. Откройте папку: backend-spring
⠀   cd C:\Users\behru\Downloads\CodeQuestApp\backend-spring

1.2. Откройте в IntelliJ IDEA как проект
     File → Open → выберите backend-spring папку

1.3. Дождитесь Gradle sync (внизу справа будет уведомление)

1.4. Нажмите Shift+F10 или Run → Run 'CodeQuestApplication'

1.5. Дождитесь запуска (увидите "Started CodeQuestApplication")

✅ Проверка: откройте http://localhost:8080/api/v1/swagger-ui.html
   Должны появиться все API endpoints с красной кнопкой "Try it out"


═══════════════════════════════════════════════════════════════════════════════
✅ ШАГ 2: ПРОВЕРЬТЕ SUPABASE ПОДКЛЮЧЕНИЕ
═══════════════════════════════════════════════════════════════════════════════

2.1. Откройте backend-spring/.env файл

2.2. Убедитесь что там:
     SUPABASE_URL=https://вашпроект.supabase.co
     SUPABASE_API_KEY=ваш_публичный_ключ

     Если нет - смотрите SUPABASE_SETUP.md для инструкций

2.3. Проверьте таблицы в Supabase:
     https://app.supabase.com → ваш проект → Table Editor
     
     Должны быть таблицы:
     ✓ profiles
     ✓ level_progress
     ✓ user_learning_profile
     ✓ xp_history
     
     ⚠️ ЕСЛИ НЕТУ: Запустите init-db.sql скрипт


═══════════════════════════════════════════════════════════════════════════════
✅ ШАГ 3: ОБНОВИТЕ ANDROID КЛИЕНТ
═══════════════════════════════════════════════════════════════════════════════

3.1. Откройте Android проект в Android Studio

3.2. Обновите URL бэкенда в ProfileViewModel.kt

     Файл: app/src/main/java/.../ui/profile/ProfileViewModel.kt
     
     🔄 НАЙДИТЕ (примерно строка 26):
        private val repository: LeaderboardRepository = 
            ApiLeaderboardRepository("http://10.0.2.2:8080/")
     
     📝 ЕСЛИ вы используете:
        • Android эмулятор + локальный ПК: http://10.0.2.2:8080/
        • Физический телефон: http://<IP_ПК>:8080/
          (найдите IP: ipconfig в cmd → IPv4 Address)
        • Cloud сервер: https://api.codequest.example.com/

3.3. Sync Gradle (File → Sync Now)

3.4. Запустите приложение (Shift+F10)

     ✓ Правильное XP и уровень

4.4. Попробуйте получить лидерборд:
     (если экран есть в приложении)

✅ ШАГ 5: ПРОВЕРЬТЕ ЧЕРЕЗ SWAGGER
═══════════════════════════════════════════════════════════════════════════════

5.1. Откройте http://localhost:8080/api/v1/swagger-ui.html

5.2. Нажмите "Try it out" у любого endpoint'а (например GET /api/v1/profiles/me)

5.3. Вам нужен Bearer токен:
     
     🔐 Получите токен в Android эмуляторе:
        • Откройте Chrome DevTools (если есть WebView)
        • Или используйте Postman (смотрите ANDROID_BACKEND_INTEGRATION.md)

5.4. Кликните "Authorize" кнопка вверху
     Введите: Bearer YOUR_TOKEN_HERE

5.5. Теперь нажимайте "Execute" у endpoints

⚠️ ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ
═══════════════════════════════════════════════════════════════════════════════

ПРОБЛЕМА: "Connection refused: 10.0.2.2:8080"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ✗ Бэкенд не запущен
   → Запустите backend (Shift+F10 в IntelliJ IDEA)
   → Проверьте что консоль показывает "Started CodeQuestApplication"

ПРОБЛЕМА: "401 Unauthorized"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ✗ Firebase токен невалиден или отсутствует
   → Убедитесь что вы залогинены в приложении
   → Проверьте google-services.json
   → Смотрите бэкенд логи для деталей

ПРОБЛЕМА: "Cannot connect to Supabase"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ✗ Supabase конфиг неправильный
   → Откройте backend-spring/.env
   → Проверьте SUPABASE_URL и SUPABASE_API_KEY
   → Смотрите https://app.supabase.com → Project Settings

ПРОБЛЕМА: "Table 'profiles' not found"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ✗ Таблицы не созданы в БД
   → Запустите SQL скрипт: backend-spring/src/main/resources/init-db.sql
   → Или используйте Supabase SQL Editor
