🔄 КАК ВСЁ РАБОТАЕТ:
═══════════════════════════════════════════════════════════════════════════════

FLOW: User Login → API Request → Backend Verification → Database Query → Response
──────────────────────────────────────────────────────────────────────────────────

1. Пользователь логинится в Android приложении
   └─→ FirebaseAuth.signInWithEmailAndPassword(email, password)
       └─→ Firebase генерирует ID Token

2. Приложение запрашивает профиль пользователя
   └─→ GET /api/v1/profiles/me
       └─→ FirebaseAuthInterceptor добавляет Authorization заголовок
           └─→ Отправляет заголовок: Authorization: Bearer <firebase_id_token>

3. Backend получает запрос с токеном
   └─→ SecurityConfig фильтр перехватывает Authorization заголовок
       └─→ Верифицирует токен через Firebase Admin SDK
           └─→ Если валиден: извлекает uid (firebase_uid)
           └─→ Если невалиден: возвращает 401 Unauthorized

4. Backend запросил данные из Supabase
   └─→ SELECT * FROM profiles WHERE id = :firebase_uid
       └─→ Если профиля нет: INSERT новый профиль со значениями по умолчанию
           └─→ SELECT вновь созданный профиль
       └─→ Если профиль есть: вернуть его

5. Backend отправляет ответ
   └─→ JSON объект { id, email, display_name, xp, level, streak, wins, avatar_url }

6. Android приложение обновляет UI
   └─→ ProfileViewModel получает данные
       └─→ Обновляет ProfileUiState
           └─→ Composable перерисовывается с новыми данными

✨ ОСОБЕННОСТИ РЕАЛИЗАЦИИ:
═══════════════════════════════════════════════════════════════════════════════

📍 AUTO-PROFILE CREATION
   Первый раз в приложении?
   → Backend автоматически создаст профиль
   → Со значениями по умолчанию: xp=0, level=1, streak=0, wins=0
   → Пользователь ничего не заметит!

📍 ATOMIC XP UPDATES
   Увеличение XP?
   → Транзакция обновляет пять операций:
      1. Обновляет xp в profiles
      2. Пересчитывает level (= floor(xp/400) + 1)
      3. Обновляет updated_at
      4. Вставляет лог в xp_history
      5. Все это происходит в одной транзакции (все или ничего)

📍 UPSERT ОПЕРАЦИИ
   Обновление прогресса уровня?
   → INSERT ... ON CONFLICT ... DO UPDATE
   → Если уровень пройден раньше: обновлять существующую запись
   → Если новый уровень: создать новую запись

📍 КАСКАДНОЕ УДАЛЕНИЕ
   Удалить пользователя?
   → Автоматически удаляются:
      • Все уровни прогресса
      • Весь контекст обучения
      • Вся история XP
   → Нет сиротских записей в БД

📋 ЧТО ДАЛЬШЕ ДЕЛАТЬ:
═══════════════════════════════════════════════════════════════════════════════

1️⃣ ШАГ 1: Запустите backend
   cd backend-spring
   ./gradlew bootRun
   (или Shift+F10 в IntelliJ IDEA)

2️⃣ ШАГ 2: Настройте Supabase URL и ключ
   Откройте backend-spring/.env
   Добавьте: SUPABASE_URL=https://ваш.supabase.co
   Добавьте: SUPABASE_API_KEY=ваш_ключ

3️⃣ ШАГ 3: Создайте таблицы в БД
   Скопируйте SQL из backend-spring/src/main/resources/init-db.sql
   Запустите в Supabase SQL Editor

4️⃣ ШАГ 4: Обновите Android URL
   Откройте ProfileViewModel.kt (строка ~26)
   Измените URL: http://10.0.2.2:8080/ (или ваш URL)

5️⃣ ШАГ 5: Запустите приложение
   Зарегистрируйтесь в приложении
   Откройте профиль
   Смотрите реальные данные из Supabase! 🎉
